# GitHub Repository Insight

Use this skill when a user wants to understand a public GitHub repository from its URL.

Always focus on technical understanding:

1. Repository purpose and use cases
2. Basic repository metadata
3. Technology stack signals
4. Important folders and files
5. Core module assumptions grounded in README and paths
6. Setup or run hints from README
7. Engineering quality signals such as tests, CI, docs, examples, and Docker
8. Potential risks or missing project information
9. Practical engineering improvement suggestions

Keep the report focused on repository understanding, technical structure, and engineering quality.
