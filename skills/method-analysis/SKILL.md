# Method Analysis Skill

Use this skill when a paper reading task requires deeper analysis of the proposed method, model, algorithm, or system.

## Workflow

1. Identify the core technical problem the method addresses.
2. Extract the method pipeline: inputs, intermediate representations, modules, outputs, and training or inference flow.
3. Distinguish the paper's own contribution from standard components borrowed from prior work.
4. Identify key assumptions, constraints, and design tradeoffs.
5. Note implementation details that affect reproducibility, such as hyperparameters, architecture choices, or preprocessing.
6. Summarize why the method is expected to improve over baselines.
7. Flag unclear method descriptions and avoid inventing missing details.

## Output Shape

Prefer concise Markdown sections:

- Method Overview
- Pipeline / Architecture
- Key Contribution
- Assumptions and Constraints
- Reproducibility Notes
- Open Technical Questions
